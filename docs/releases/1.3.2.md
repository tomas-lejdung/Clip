# Clip 1.3.2

Clip 1.3.2 improves Live Share viewer reliability and corrects Retina window sizing.

## Live Share

- Continue gathering usable ICE candidates when one STUN server reports a
  per-server timeout, while still failing when the peer connection itself fails.
- Prevent native viewer windows from entering an invalid fullscreen/tab state
  that could crash the viewer.
- Keep Retina captures encoded at full pixel resolution while presenting them
  at the source Mac's logical window size in both native and web viewers.
- Preserve logical source geometry across reconnects, codec changes, and legacy
  hosts that do not send the new optional metadata.

The matching `1.3.2` Live Share server image updates the hosted web viewer. The
signaling and media routing protocol remains backward compatible.

## Requirements

Clip requires macOS 15 or later on Apple silicon. This build uses the same
stable Apple Development identity as previous updater releases.
