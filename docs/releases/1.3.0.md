# Clip 1.3.0

Clip 1.3.0 adds Live Share, persistent Friends, and a native multi-window
viewer while keeping recordings and sharing as separate workflows.

## Live Share

- Enter a preparation screen before anything is captured, choose an optional
  access code, copy a complete invite, or join an invite from another Clip
  user.
- Share up to four application windows, one automatically followed focused
  window, or a fullscreen display.
- Choose VP8, H.264, VP9, or AV1 at runtime, plus quality, frame rate, focused
  source priority, and performance or quality encoding mode.
- Optionally share application-scoped or fullscreen system audio through one
  Opus track. Microphone audio is never captured by Live Share.
- Inspect viewers, active streams, codec, resolution, frame rate, actual and
  target bitrate, connection route, delay, drops, and limiting reason.
- Continue using the encrypted browser viewer as a fallback.

## Friends and native viewing

- Add another Clip installation as a Friend only after both sides complete a
  signed P-256 pairing exchange.
- See saved Friends when they are sharing and request to join without asking
  them to resend a complete invite.
- Hosts explicitly allow or deny every native Friend connection. The server
  stores no Friend graph, password, private identity, SDP, ICE candidate, or
  media payload.
- View up to four remote sources as independent native macOS windows with a
  stable colored identity border, Retina-aware Actual/Fit sizing, remote cursor
  mapping, mute and volume controls, and P2P/TURN statistics.
- Auto mode keeps one stable viewer window and switches its contents to the
  host's focused source without stealing local keyboard focus.
- Pairing and trust changes use crash-safe receipts so an interrupted setup
  cannot silently create a one-sided Friend relationship.

## Reliability

- Fix secure device identity creation in directly signed builds by using the
  standard macOS Keychain without requiring a provisioning profile.
- Report identity-storage failures separately from room or network failures.
- Bound signaling, receiver tracks, queues, and reconnect handoff so malformed
  or overloaded sessions fail visibly instead of accumulating hidden work.

## Requirements and first launch

Clip requires macOS 15 or later on Apple silicon.

This build remains Apple Development signed rather than Developer ID signed and
notarized. A direct browser download may therefore require **Open Anyway** in
**System Settings → Privacy & Security**.
