# Clip 1.3.3

Clip 1.3.3 fixes native Friend joining and makes the **Actual** viewer size
consistent across Retina and non-Retina Macs.

## Live Share

- Reserve each encrypted Friend-join message sequence before sending it, so
  concurrent WebRTC offer and ICE messages cannot reuse a sequence number and
  be rejected as an invalid native rendezvous message.
- Keep a route closed if it disconnects while approval is being sent instead
  of restoring stale connection state afterward.
- Define **Actual** using the source Mac's logical window size rather than the
  viewer display's backing scale. Retina-hosted windows therefore no longer
  appear doubled on non-Retina viewers, and external-display windows no longer
  appear halved on Retina viewers.
- Preserve the existing decoded-pixel fallback when viewing a stream from an
  older host that does not provide logical source dimensions.

This is an app-only update. The Live Share server and web viewer are unchanged.

## Requirements

Clip requires macOS 15 or later on Apple silicon. This build uses the same
stable Apple Development identity as previous updater releases.
