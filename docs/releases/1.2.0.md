# Clip 1.2.0

Clip 1.2.0 adds native Live Share: share application windows or a fullscreen
display with browser viewers without creating a recording.

## Live Share

- Start a room from the menu-bar popover and copy its browser link.
- Share up to four application windows, or switch to one fullscreen display.
- Use the focused-window Share/Stop control and the fixed status HUD without
  granting Accessibility or Automation access.
- Choose a total quality ceiling, 15 or 30 FPS, focused-window prioritization,
  automatic focused-window sharing and a codec preference. VP8 is the default;
  H.264 and VP8 are exact choices, while VP9 falls back to VP8 and AV1 falls
  back through VP9 to VP8 when the connected runtime cannot provide the newer
  codec. Codec changes renegotiate connected viewers without replacing the
  room. Sixty FPS appears only when the runtime supports it.
- See connected viewers, connection route, active streams, codec, actual versus
  target/ceiling bitrate, frame rate, resolution, encode/send delay, drops,
  limiting reason, and elapsed time.
- Share 5K and 6K sources at native resolution with VP8. H.264 automatically
  preserves aspect ratio inside the hardware encoder's safe large-frame limit.
  Smaller odd-sized windows stay pixel-exact through capture and lose only the
  unavoidable final H.264 alignment row or column instead of being rescaled.
- Keep latency current with bounded capture/encoder queues, stale-frame drops,
  static-frame keyframe recovery, selected-stream playback-buffer reporting in
  the browser viewer, and a visible aggregate H.264 freshness-drop diagnostic.
- Keep H.264 text sharp with VideoToolbox quality 0.98, soft live rate control,
  no encoder-side hard data-rate cap, and a hardware callback deadline that does
  not collapse to 33 ms when 60 FPS is selected.
- Recover from bounded signaling interruptions without writing live frames to
  History or disk.
- Optionally share application-scoped or Fullscreen system audio through one
  Opus track. The browser viewer provides mute, volume, and autoplay-unlock
  controls; microphone audio is never captured.

Live Share uses Clip's Go service for room advertisement and bounded encrypted
signaling, then sends media through encrypted WebRTC transport. The optional
access code is checked by the host and is not disclosed to the service. Live
Share does not include remote control.

## Requirements and first launch

Clip requires macOS 15 or later on Apple silicon.

This build remains Apple Development signed rather than Developer ID signed and
notarized. A direct browser download may therefore require **Open Anyway** in
**System Settings → Privacy & Security**.
